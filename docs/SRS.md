# Software Requirements Specification (SRS)

## 1. Introduction
### 1.1 Purpose
This document describes the functional and non-functional requirements of the TeamTrack system.

### 1.2 Scope
TeamTrack is a web-based issue and task management system designed for small software teams.

### 1.3 Definitions
- Issue: A task or bug tracked in the system
- Sprint: A time-boxed development period
- User: A registered system member

## 2. Overall Description
### 2.1 User Classes
- Admin
- Team Member

### 2.2 System Environment
- Web application
- Client-server architecture

## 3. Functional Requirements
- FR-01: User shall be able to log in to the system
- FR-02: Admin shall be able to create projects
- FR-03: User shall be able to create and update tasks

## 4. Non-Functional Requirements
- NFR-01: The system shall be secure and require authentication
- NFR-02: The system shall be responsive and user-friendly
